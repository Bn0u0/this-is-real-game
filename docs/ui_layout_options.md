# 主介面排版結構提案 (Layout Structure Options)

此提案**不討論美術風格 (Style)**，僅專注於**「按鈕放在哪裡」**與**「如何導航」**。
目標是確立 MVP1 的功能性排版 (Layout Strategy)。

---

## 方案 1: 沉浸式場景 (Immersive World / Hub & Spoke)
**這是目前的開發方向。**
將「按鈕」直接整合進遊戲場景的物體中，螢幕邊緣只留最必要的資訊。

### 排版邏輯
*   **中央區域**: 空無一物 (留給 3D 角色與場景)。
*   **導航方式**:
    *   **點擊場景物體**: 點擊「箱子」打開裝備，點擊「角色」打開狀態。
    *   **返回**: 點擊空白處或角落的 [返回] 按鈕。
*   **角落 HUD**:
    *   **左上**: 玩家頭像、等級。
    *   **右上**: 貨幣 (資源)。
    *   **右下**: **巨大的 [出擊] 按鈕**。

### 適合 MVP1 的原因
*   邏輯簡單：所見即所得。
*   最大限度展示遊戲畫面。

### 缺點
*   功能變多時，場景會塞不下太多物體。

---

## 方案 2: 底部導航欄 (Bottom Navigation Dock)
**經典手遊/App 排版 (如：原神、絕地求生 M)。**
將所有主要功能歸納在螢幕底部的一排固定按鈕。

### 排版邏輯
*   **底部區域**: 固定一條高約 15% 的導航欄。
    *   [ 主頁 ] [ 角色 ] [ 裝備 ] [ 商店 ]
*   **出擊按鈕**: 通常採用「懸浮按鈕 (Floating Action Button)」形式，疊在導航欄上方或位於右下角。
*   **中央區域**: 根據底部 Tab 切換顯示內容 (切到裝備就顯示裝備介面)。
*   **頂部區域**: 僅顯示資源與設置。

### 適合 MVP1 的原因
*   **擴充性最強**：新增功能只需在底部加一個圖標。
*   玩家非常熟悉這種操作邏輯 (Tab Switching)。

### 缺點
*   螢幕下方會被佔用一塊區域，場景沉浸感稍弱。

---

## 方案 3: 側邊列表/儀表板 (Sidebar / Dashboard)
**RPG / 策略遊戲常見排版 (如：明日方舟、戰雙)。**
將功能選單收納在螢幕的一側 (通常是右側)，形成一個明確的操作面板。

### 排版邏輯
*   **左側 (2/3)**: 角色展示區 (3D View)。
*   **右側 (1/3)**: 垂直排列的功能選單。
    *   按鈕由上而下排列：[出擊]、[角色]、[裝備]、[任務]...
*   **操作流**: 左手負責切換角色/旋轉視角，右手負責點擊功能。

### 適合 MVP1 的原因
*   **資訊密度高**：可以一次列出很多功能按鈕而不顯得亂。
*   非常適合橫屏 (Landscape) 遊戲的操作人體工學。

### 缺點
*   畫面左右重心不平衡 (需靠美術調整)。

---

## 開發者建議 (Developer Recommendation)

針對目前 **MVP1 跑通流程** 的目標：

*   **如果您希望「快」且「直覺」**：選擇 **方案 2 (底部導航欄)**。因為這是標準 UI 組件，不管是寫代碼還是用 Canva 拼圖都最容易模組化。
*   **如果您希望「酷」且「延續現有 3D 場景」**：維持 **方案 1 (沉浸式場景)**，但我們需要明確定義每個角落的 HUD 規範。

請問您決定採用哪種 **「排版結構」**？ (1, 2, 或 3)
